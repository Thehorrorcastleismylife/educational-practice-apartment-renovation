база 1 users:
 содержит основную информацию о пользователе. отдельная база админ не нужна, так как админ не хранит данных, отличных от данных пользователя. pk user_id нужен, чтобы при выполнении заказа ссылаться на него и индентифицировать его как уникальный. связи: один ко многим с таблицей оредрс
база 2 remont_services:
 содержит данные услуги, путь(path_to_media) нужен для изменения его через админ панель. id услуги - pk, который нужен, чтобы в ордере ссылаться на конкретную услугу. связи: много к одному с таблицей ордерс.э
база 3 orders:
 хранит данные для заказа, pk order_id нужен для индентификации заявки как уникальной и нужен для получения данных о ордере для конкретного id из кода sql. ссылается на поля id услуги и id пользователя. связи много-к-одному с users и много-к-одному с remont_services.

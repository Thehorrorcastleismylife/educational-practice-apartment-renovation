база 1 users:
 содержит основную информацию о пользователе. отдельная база админ не нужна, так как админ не хранит данных, отличных от данных пользователя. pk user_id нужен, чтобы при выполнении заказа ссылаться на него и индентифицировать его как уникальный. Связь: один ко многим с orders (один пользователь может сделать много заказов).
база 2 remont_services:
 содержит данные услуги, путь(path_to_media) нужен для изменения его через админ панель. id услуги - pk, который нужен, чтобы в ордере ссылаться на конкретную услугу. Связь: один ко многим с orders (одна услуга может быть в многих заказах).
база 3 orders:
 хранит данные для заказа, pk order_id нужен для индентификации заявки как уникальной и нужен для получения данных о ордере для конкретного id из кода sql. ссылается на поля id услуги и id пользователя. Поля order_phone и order_address копируются при создании заказа для сохранения истории. Связи: многие-к-одному с users и многие-к-одному с remont_services.

# Маршрут гостя
Гость открывает главную страницу и просматривает слайдер с примерами работ и общую информацию о компании. Далее гость переходит в каталог услуг через навигацию или призыв к действию на главной. В каталоге гость использует поиск или фильтры для нахождения нужной услуги, затем раскрывает детальную информацию об интересующей услуге, чтобы ознакомиться с ценой, сроками и условиями. Для того чтобы создать заявку, гость должен авторизоваться, поэтому он переходит на страницу входа и регистрации. После заполнения формы регистрации и создания аккаунта гость автоматически переходит на ту же страницу услуги

Полный маршрут: Главная → Каталог услуг → Детальная информация об услуге → Страница входа и регистрации → Форма создания заявки

# Маршрут авторизованного пользователя

Авторизованный пользователь начинает с главной страницы или сразу переходит в каталог услуг. В каталоге он использует поиск и фильтры для подбора нужной услуги, раскрывает детальную информацию для просмотра цены и сроков. Убедившись в подходящих условиях, пользователь создает на странице услуги заявку на ремонт. После отправки заявки пользователя перекидывает в личный кабинете, где может просмотреть список всех своих заявок и их статусы.

Полный маршрут: Главная → Каталог услуг → Детальная информация об услуге → Создание заявки → Просмотр своих заявок в личном кабинете

# Маршрут администратора
Администратор входит в систему через страницу авторизации и попадает сразу в админ-панель. В админ-панели администратор видит инструменты управления услугами и заявками. Для публикации новой услуги администратор заполняет форму создания услуги, указывая название, цену, фото, время работы и дополнительные условия, после чего услуга появляется в каталоге. При необходимости администратор просматривает список существующих услуг и удаляет устаревшие позиции. Также администратор просматривает заявки пользователей, полученные через личный кабинет, и обрабатывает их — связывается с мастером по ремонту и передаёт контактные данные пользователя для организации работ. Все действия администратора выполняются в рамках одной админ-панели без переходов на другие страницы.

Полный маршрут: Страница входа и регистрации → Админ-панель → Создание новой услуги → Просмотр списка услуг → Удаление услуги → Просмотр заявок пользователей → Обработка заявки и передача данных мастеру и пользователю

Альтернативный маршрут: Админ-панель → Просмотр заявок пользователей → Просмотр списка услуг → Возврат к просмотру заявок

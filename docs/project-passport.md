# Паспорт проекта

## Название проекта
Ремонт квартир

## Предметная область
Предоставление услуг по ремонту квартир. целевая аудитория – люди, которые хотят получить услуги по ремонту квартир(имеется квартира), разных возрастов(18-100)
На сайте будут размещаться данные о названии услуги, цене ремонта, пример работы, контакт для связи с работником(оператором)

## Проблема
Пользователю неудобно узнавать информацию о ремонтных услугах: приходится долго искать предложения у знакомых или на разных площадках, не специализированных под ремонтные услуги. Отсутствует единый каталог услуг, а другие сайты могут не предоставлять гарантий.
основное действие пользователя – заказать услугу по ремонту квартиры
администратор сможет публиковать новые услуги
может просматривать примеры работ, узнать цену, сделать поиск по сайту, создать заявку на ремонт квартиры 


## Цель проекта
Разработать веб‑сайт по услугам ремонта квартир, позволяющий пользователям просматривать услуги, выполнять поиск, а администратору — управлять услугами и обрабатывать заявки пользователей.

## Целевая аудитория
- новые клиенты, которые хотят получить услуги ремонта (владельцы квартир);
- клиенты, которым порекомендовали услуги другие клиенты;
- клиенты, пришедшие из интернет‑рекламы.

## Роли пользователей
- **Гость**: просмотр списка услуг, поиск и фильтрация;
- **Пользователь**: создание заявки на ремонт, просмотр своих заявок, связь со службой поддержки, выход из системы;
- **Администратор**: создание новой услуги, просмотр и удаление услуг, просмотр заявок пользователей.

## Контентная сущность
Услуга.

## Операционная сущность
Заявка на ремонт.

## Основные функции

**Для гостя:**
- просмотр главной страницы;
- просмотр каталога услуг;
- просмотр детальной страницы услуги (цена, фото, время на работу, дополнительные условия);
- регистрация;
- авторизация.

**Для пользователя:**
- создание заявки на ремонт;
- просмотр своих заявок;
- выход из системы.

**Для администратора:**
- создание новой услуги;
- просмотр созданных услуг;
- удаление услуги;
- просмотр заявок пользователей.

## Данные, которые необходимо хранить
- пользователи;
- роли пользователей;
- услуги;
- заявки;
- статусы заявок.

## Планируемые страницы
- главная страница;
- окно входа/регистрации;
- личный кабинет;
- каталог;
- админ‑панель;
- форма создания заявки;
- страница услуги.

## Предполагаемое дизайн‑направление
сайт будет сделан в светлых цветах, на главной странице будет слайдер с работами по ремонту

## Ожидаемый результат
Готовый веб‑сайт с функционалом для:
- просмотра и поиска услуг по ремонту квартир (для гостей и пользователей);
- создания и отслеживания заявок на ремонт (для пользователей);
- управления услугами и обработки заявок (для администратора).
